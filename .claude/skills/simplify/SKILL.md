# Simplify Skill

Review changed code and refactor to eliminate duplication, improve quality, and increase efficiency.

## When to Use
- After writing new code
- When code review reveals opportunities for improvement
- Before committing changes

## Review Criteria
1. **DRY** — Is there code duplication that should be extracted?
2. **Readability** — Is the code clear and self-documenting?
3. **Efficiency** — Any unnecessary complexity or redundant operations?
4. **Best Practices** — Does it follow project conventions?

## Output
- List of issues found
- Refactored code with improvements
- Explanation of each change
