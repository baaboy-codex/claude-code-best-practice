# Debug Skill

Debug failing commands or code issues by diagnosing root causes and providing solutions.

## When to Use
- A command fails with an error
- Code is not working as expected
- Build errors occur
- Tests are failing

## Process
1. **Reproduce** — Run the failing command to see the error
2. **Analyze** — Examine error messages and stack traces
3. **Hypothesize** — Identify possible root causes
4. **Test** — Try fixes incrementally
5. **Verify** — Confirm the fix works

## Common Debug Patterns
- Missing dependencies
- Environment variable issues
- Path/import problems
- Type errors
- Race conditions
- Memory leaks

## Output
- Root cause analysis
- Step-by-step fix
- Verification steps
