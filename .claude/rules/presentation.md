# Presentation Rules

## Voice
- Use semi-formal language
- Short, punchy sentences preferred
- Active voice throughout

## Formatting
- Use markdown headers (##, ###)
- Use tables for structured comparisons
- Use code blocks for examples
- Use bullet points sparingly

## Content Structure
1. Lead with the most important information
2. Use progressive disclosure (general → specific)
3. End with actionable takeaways

## Code Examples
- Show complete, runnable code
- Include comments explaining key parts
- Show bad → good transformations when helpful
