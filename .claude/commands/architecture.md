# Architecture Analysis Command

Analyze the project structure and document the architecture.

## Steps
1. List all source files and directories
2. Identify key components and their relationships
3. Look for patterns (MVC, layered, microservices, etc.)
4. Document the data flow
5. Identify external dependencies

## Output Format
- **Overview**: High-level architecture description
- **Components**: Key modules and their responsibilities
- **Data Flow**: How data moves through the system
- **Dependencies**: External libraries and services
- **Diagrams**: ASCII architecture diagrams when helpful
